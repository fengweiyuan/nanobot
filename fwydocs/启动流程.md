# Nanobot 启动流程详解

## 概述

nanobot 是一个 CLI 工具，通过 **Typer** 框架（基于 Click）构建。入口点不是单一的 `main.py`，而是通过 **`nanobot/__main__.py`** 和 **`nanobot/cli/commands.py`** 协作。

---

## 启动入口

### 方式 1：安装后直接运行（推荐）

```bash
# 方式 A：使用 nanobot 命令（已安装到系统）
nanobot agent
nanobot gateway
nanobot status

# 方式 B：使用 Python 模块方式
python -m nanobot agent
python -m nanobot gateway
```

### 方式 2：开发时直接运行

```bash
# 从源代码根目录运行
cd /Users/fwy/code/pub/nanobot
python -m nanobot agent
```

### 安装配置

**文件**：`pyproject.toml` 行 59-60

```toml
[project.scripts]
nanobot = "nanobot.cli.commands:app"
```

这行配置告诉 pip：当用户运行 `nanobot` 命令时，调用 `nanobot.cli.commands` 模块中的 `app` 对象。

### 1. `nanobot onboard` —— 首次初始化

**文件**：`commands.py` 行 157-196

```python
@app.command()
def onboard():
    """Initialize nanobot configuration and workspace."""

    # 1. 创建或刷新配置
    config_path = get_config_path()  # ~/.nanobot/config.json
    if not config_path.exists():
        save_config(Config())
        console.print("✓ Created config at ~/.nanobot/config.json")

    # 2. 创建工作空间
    workspace = get_workspace_path()  # ~/.nanobot/workspace/
    workspace.mkdir(parents=True, exist_ok=True)
    console.print("✓ Created workspace at ~/.nanobot/workspace/")

    # 3. 同步模板文件
    sync_workspace_templates(workspace)
    # 创建必要的文件：
    # - HEARTBEAT.md (周期任务)
    # - memory/MEMORY.md (持久化记忆)
    # - memory/HISTORY.md (对话历史)
    # - skills/ (技能目录)
```

**流程**：
```

    ↓
生成 ~/.nanobot/config.json（空默认配置）
    ↓
生成 ~/.nanobot/workspace/（工作目录）
    ↓
初始化模板文件
    ↓
提示用户添加 API key
```

### 3. `nanobot gateway` —— 启动网关（多渠道）

**文件**：`commands.py` 行 245-404

这是最复杂的启动模式，启动完整的 agent 系统。

#### 3.1 初始化流程

```python
@app.command()
def gateway(port: int = 18790, verbose: bool = False):
    """Start the nanobot gateway."""

    # 1. 加载配置
    config = load_config()  # ~/.nanobot/config.json

    # 2. 创建 Message Bus（所有组件的通信枢纽）
    bus = MessageBus()

    # 3. 创建 LLM 提供商
    provider = _make_provider(config)

    # 4. 创建 Session Manager（管理多个用户的对话）
    session_manager = SessionManager(config.workspace_path)

    # 5. 创建 Cron 服务（计划任务）
    cron = CronService(cron_store_path)

    # 6. 创建 Agent Loop（核心处理引擎）
    agent = AgentLoop(
        bus=bus,
        provider=provider,
        workspace=config.workspace_path,
        model=config.agents.defaults.model,
        temperature=config.agents.defaults.temperature,
        max_tokens=config.agents.defaults.max_tokens,
        max_iterations=config.agents.defaults.max_tool_iterations,
        memory_window=config.agents.defaults.memory_window,
        brave_api_key=config.tools.web.search.api_key,
        exec_config=config.tools.exec,
        cron_service=cron,
        restrict_to_workspace=config.tools.restrict_to_workspace,
        session_manager=session_manager,
        mcp_servers=config.tools.mcp_servers,
        channels_config=config.channels,
    )

    # 7. 设置 Cron 回调
    async def on_cron_job(job):
        response = await agent.process_direct(job.payload.message, ...)
        if job.payload.deliver:
            await bus.publish_outbound(...)
    cron.on_job = on_cron_job

    # 8. 创建 Channel Manager（所有聊天平台）
    channels = ChannelManager(config, bus)

    # 9. 创建 Heartbeat Service（周期任务）
    heartbeat = HeartbeatService(config.workspace_path, agent)
```

#### 3.2 网关启动流程图

```
nanobot gateway
    ↓
[Config] 加载 config.json
    ↓
[Bus] 创建消息队列
    ↓
[Provider] 创建 LLM 连接
    ↓
[Agent Loop] 初始化核心引擎
    ↓
[Channels Manager] 初始化所有渠道
    │
    ├─ TelegramChannel (if enabled)
    ├─ DiscordChannel (if enabled)
    ├─ WhatsAppChannel (if enabled)
    ├─ FeishuChannel (if enabled)
    ├─ SlackChannel (if enabled)
    ├─ EmailChannel (if enabled)
    ├─ DingTalkChannel (if enabled)
    ├─ MatrixChannel (if enabled)
    ├─ QQChannel (if enabled)
    └─ MockchatChannel (if enabled)
    ↓
[Cron Service] 初始化计划任务
    ↓
[Heartbeat Service] 初始化周期任务
    ↓
启动所有异步任务：
├─ agent_loop.run()        # Agent Loop 主循环
├─ channels.start()        # 所有渠道开始监听
├─ heartbeat.start()       # 周期任务检查
├─ cron.start()           # 计划任务服务
└─ 其他后台服务
    ↓
系统运行，等待用户消息...
    ↓
用户从任何渠道发送消息 → InboundMessage → Bus
    ↓
Agent Loop 处理 → OutboundMessage → Bus
    ↓
相应渠道发送回复给用户
    ↓
(Ctrl+C) 关闭所有任务，优雅退出
```

## 配置加载流程

**文件**：`config/loader.py`

```python
def get_config_path() -> Path:
    """获取配置文件路径"""
    return Path.home() / ".nanobot" / "config.json"

def load_config() -> Config:
    """
    加载配置
    1. 读取 ~/.nanobot/config.json
    2. Pydantic 解析和验证
    3. 应用默认值
    """
    config_path = get_config_path()
    if not config_path.exists():
        raise FileNotFoundError(f"Config not found at {config_path}")

    with open(config_path) as f:
        config_data = json.load(f)

    return Config.model_validate(config_data)
```

## Provider 工厂函数

**文件**：`commands.py` 行 202-244

```python
def _make_provider(config: Config):
    """根据配置创建对应的 LLM 提供商"""

    model = config.agents.defaults.model
    provider_name = config.get_provider_name(model)
    p = config.get_provider(model)

    # 1. OpenAI Codex (OAuth)
    if provider_name == "openai_codex":
        return OpenAICodexProvider(default_model=model)

    # 2. Custom (直接 OpenAI 兼容端点，不走 LiteLLM)
    if provider_name == "custom":
        return CustomProvider(
            api_key=p.api_key,
            api_base=p.api_base,
            default_model=model,
        )

    # 3. LiteLLM (支持 20+ 提供商)
    return LiteLLMProvider(
        api_key=p.api_key,
        api_base=p.api_base or None,
        default_model=model,
        extra_headers=p.extra_headers or {},
        provider_name=provider_name,
    )
```

## 关键数据结构

### InboundMessage (用户消息)

```python
@dataclass
class InboundMessage:
    channel: str              # "telegram", "discord", "cli"
    sender_id: str            # 用户 ID
    chat_id: str              # 聊天 ID
    content: str              # 消息内容
    session_key: str          # "{channel}:{chat_id}"
    metadata: dict | None = None
    media: list[str] | None = None  # 图片路径
```

### OutboundMessage (机器人回复)

```python
@dataclass
class OutboundMessage:
    channel: str              # 目标渠道
    chat_id: str              # 目标聊天 ID
    content: str              # 回复内容
    metadata: dict | None = None
    media: list[str] | None = None  # 要发送的文件
```

## 总结

| 命令 | 用途 | 运行模式 | 关键函数 |
|------|------|---------|---------|
| `nanobot onboard` | 首次初始化 | 同步 | `onboard()` |
| `nanobot agent -m "..."` | 一次性问答 | 异步 | `agent()` + `process_direct()` |
| `nanobot agent` | 交互式聊天 | 异步 + 并发 | `agent()` + `run_interactive()` |
| `nanobot gateway` | 多渠道网关 | 异步 + 并发 | `gateway()` + 完整系统 |
| `nanobot status` | 查看状态 | 同步 | `status()` |
| `nanobot cron *` | 管理任务 | 同步 | `cron_*()` |
| `nanobot channels *` | 管理渠道 | 同步/异步 | `channels_*()` |

**核心架构**：
```
__main__.py → commands.py (Typer app) → 各个命令函数
                                        → 初始化各个核心组件
                                        → 启动异步任务
                                        → 持续运行或一次性执行
```
